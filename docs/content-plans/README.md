# Metaverse Content Plans

ไฟล์ในโฟลเดอร์นี้เป็นแผนคอนเทนต์ของเพจ Metaverse Resolution

## โครงสร้าง

- `generated/` ไฟล์แผนที่สร้างจากสคริปต์ เช่น `.md`, `.csv`, `.json`
- `references/` ไอเดียและแนวทางที่ใช้เป็นข้อมูลตั้งต้น
- `archive/` แผนเก่าที่เก็บไว้ดูย้อนหลัง

## สร้างแผนเดือนใหม่แบบสลับโปรดักต์

```bash
npm run content:metaverse -- --start 2026-08-01 --end 2026-08-31 --out docs/content-plans/generated --prefix metaverse-content-2026-08
```

## สร้างแผนและ sync เข้า Planner

```bash
npm run content:metaverse -- --start 2026-08-01 --end 2026-08-31 --out docs/content-plans/generated --prefix metaverse-content-2026-08 --sync
```

กติกาหลักของสคริปต์:

- ลง feed ตามจังหวะ 3 โพสต์ต่อสัปดาห์: พุธ, ศุกร์, อาทิตย์ เวลา 18:00 น.
- เว้นจันทร์และอังคารเป็นหลัก เพื่อให้ทีมมีเวลาผลิตและตรวจงานมากขึ้น
- ระยะห่างโพสต์เป็น 2-2-3 วัน เพื่อให้เพจไม่แน่นเกินไปและยังสม่ำเสมอ
- Story เป็นพื้นที่เสริมเท่านั้น และจะถูกสร้างเมื่อรันด้วย `--include-stories` (ตั้งไว้วันเสาร์)
- สลับ product เพื่อลดหัวข้อใกล้กันเกินไป
- `ประจำเดือน` ใช้ได้เดือนละ 1 ครั้ง และใช้กับโพสต์รวม product ต้นเดือนเท่านั้น
- ทุกเดือนเต็มควรครอบคลุม product หลักครบทุกตัว
- ตัวจัดตารางอัตโนมัติเป็นค่าเริ่มต้นแล้ว ใช้ `--no-auto` เฉพาะเมื่อต้องการกลับไปใช้แผนเขียนมือเดิม

หมายเหตุ: ไฟล์ใน `generated/` และ `archive/` บางไฟล์เป็น snapshot เก่าก่อนปรับ cadence จึงอาจยังมีตารางแบบ 4 โพสต์ต่อสัปดาห์หรือวันโพสต์เดิม ให้ยึดกติกาในเอกสารนี้และ `scripts/generate-metaverse-content.mjs` เป็นปัจจุบัน
